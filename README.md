# Godot-SRPG-DEMO

一个基于Godot引擎的回合制SRPG游戏演示项目，展示了回合制战斗、网格地图、敌人AI等核心SRPG元素。

## 功能特性

- **回合制战斗系统**：基于速度属性的行动顺序，支持轮数计数
- **职业系统**：包含哥布林（近战）、精灵弓手（远程）、战争天使（精英）、骷髅士兵（近战，属性更强）、生命法师（治疗型，移动较慢）和战士（玩家单位）六种职业
- **运动轨迹**：所有角色移动时显示路径动画，对角移动拆分为两段直线
- **攻击范围**：显示攻击范围动画，角色中心位置不涂色
- **地图系统**：棋盘式网格布局，清晰的格子显示和障碍物系统
- **AI系统**：敌人根据血量调整攻击欲望，优先攻击玩家
- **精英敌人AI**：精英敌人具备近战和远程攻击能力，根据血量和距离智能选择攻击方式
- **调试系统**：包含敌人生成器和属性编辑器，方便测试
- **装备系统**：单位可以装备武器和防具，提升属性
- **背包系统**：单位拥有背包，可以存放和管理物品
- **单位详情页**：右键点击单位可以查看详细属性、装备和背包信息，玩家单位可以在详情页更换装备

## 技术栈

- Godot 4.6.1
- C#
- .NET 8.0

## 如何运行

1. 确保安装了Godot 4.6.1或更高版本
2. 打开Godot引擎
3. 导入项目文件夹
4. 点击运行按钮启动游戏

## 游戏操作

- 选择玩家角色：点击玩家单位
- 移动：点击高亮的格子
- 攻击：选择攻击类型后点击范围内的敌人
- 结束回合：点击"结束回合"按钮
- 查看单位详情：右键点击任意单位，打开单位详情页
- 更换装备：在玩家单位的详情页中，点击背包中的装备可以进行更换

## 项目结构

### 核心文件
- `src/GameManager.cs`：游戏主逻辑，负责初始化各个模块和游戏循环
- `src/Unit.cs`：单位属性和行为定义，包含装备和背包系统
- `src/TurnManager.cs`：回合管理系统
- `src/Grid.cs`：地图网格系统
- `src/Obstacle.cs`：障碍物系统

### 模块结构

#### AI模块
- `src/AI/EnemyAI.cs`：敌人AI逻辑，包括移动和攻击决策

#### 公共模块
- `src/Common/Constants.cs`：常量定义

#### 物品模块
- `src/Items/Equipment.cs`：装备类定义，包含武器和防具
- `src/Items/Item.cs`：物品基类定义
- `src/Items/Inventory.cs`：背包系统，管理物品存储

#### 管理器模块
- `src/Managers/DebugManager.cs`：调试系统，包含敌人生成器和属性编辑器
- `src/Managers/GameStateManager.cs`：游戏状态管理，检测战斗结束条件
- `src/Managers/InputManager.cs`：输入处理，处理玩家点击和操作，包括右键打开单位详情页
- `src/Managers/MapManager.cs`：地图管理，负责地图生成和障碍物管理
- `src/Managers/UIManager.cs`：UI管理，负责游戏结束菜单和重启功能
- `src/Managers/UnitDetailUI.cs`：单位详情页UI，显示单位属性、装备和背包信息
- `src/Managers/UnitManager.cs`：单位管理，负责单位的创建、绘制和更新
- `src/Managers/UnitRenderer.cs`：单位渲染，负责单位和血条的显示

#### 系统模块
- `src/Systems/CombatRenderer.cs`：战斗渲染，负责攻击范围和动画显示
- `src/Systems/CombatSystem.cs`：战斗系统，处理攻击逻辑和伤害计算
- `src/Systems/MovementSystem.cs`：移动系统，处理单位移动和路径动画

## 敌人AI行为

- **普通敌人**（哥布林、精灵弓手）：根据血量和距离决定攻击或移动
- **骷髅士兵**：近战攻击，属性比哥布林更强
- **生命法师**：优先治疗范围内的低血量敌人，移动较慢，尽可能远离玩家
- **战争天使**（精英）：
  - 距离玩家1格且血量≥50%：使用近战攻击
  - 距离玩家1格且血量<50%：尝试撤退到远程位置后使用远程攻击
  - 距离玩家>1格：根据距离和血量决定攻击或移动

## 调试功能

- **敌人生成器**：可以生成不同职业的敌人
- **单位属性编辑**：可以调整单位的各项属性
- **触发游戏失败**：可以直接触发游戏失败，方便测试重启功能

## 最近修复

- **数据持久化**：将装备和单位数据从硬编码迁移到JSON文件，实现数据与逻辑分离
- **修复类型转换错误**：添加安全的类型检查和转换方法，解决Godot.Variant类型转换异常
- **修复空引用异常**：在关键方法中添加空检查，提高代码稳定性
- **优化敌人生成**：从JSON文件加载敌人属性，确保属性一致性
- **修复调试菜单**：确保敌人生成器只显示和生成非玩家单位
- **修复重新开始游戏时的UI问题**：解决了游戏结束后点击重新开始时弹窗在地图底下的问题，以及重新开始后UI与初始启动不一致的问题
- **优化攻击系统**：改进了攻击范围显示和攻击逻辑，确保远程攻击和近战攻击的正确处理
- **优化AI行为**：改进了敌人AI策略，现在敌人可以在移动后检查并执行攻击，提高了AI的攻击性和挑战性
- **UI代码优化**：简化了UnitDetailUI中的按钮创建逻辑，统一了关闭按钮的处理，减少了重复代码
- **物品加载逻辑改进**：在UnitManager中实现了新的物品加载逻辑，支持字典格式数据，同时保持向后兼容性

## 代码架构

项目采用模块化设计，遵循以下原则：

1. **单一职责原则**：每个类只负责一个功能
2. **依赖注入**：通过构造函数注入依赖，减少耦合
3. **事件驱动**：使用事件系统处理游戏状态变化
4. **分层设计**：将游戏逻辑分为不同层次，便于维护和扩展

## 扩展开发

### 添加新功能

1. **添加新职业**：
   - 在`Unit.cs`中添加新的职业类型
   - 在`EnemyAI.cs`中添加对应职业的AI逻辑

2. **添加新系统**：
   - 在`src/Systems/`目录下创建新的系统类
   - 在`GameManager.cs`中初始化和集成新系统

3. **扩展AI行为**：
   - 修改`EnemyAI.cs`中的决策逻辑
   - 可以添加更复杂的AI行为，如团队协作、战术选择等

4. **添加新UI元素**：
   - 在`UIManager.cs`中添加新的UI组件
   - 确保UI元素的创建和销毁逻辑正确

### 代码规范

- 类名使用PascalCase命名法
- 方法和属性使用PascalCase命名法
- 字段使用camelCase命名法
- 常量使用ALL_CAPS命名法
- 每个类和方法都应该有清晰的注释
